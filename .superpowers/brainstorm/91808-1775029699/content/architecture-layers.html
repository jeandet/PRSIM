Key design decisions shown

  • Window is an abstract interface in core — no SDL types leak into core headers
  • Backend is the window factory — create_window(WindowConfig) returns Window&
  • Backend owns window lifetime — windows live as long as the backend (or until explicitly closed)
  • Events get a WindowId — ready for multi-window routing
  • RenderConfig stays on backend — font is a rendering concern, not a window concern
